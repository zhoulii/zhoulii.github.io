02.原因说明

上述案例之所以会发生字段类型不匹配，原因在于 Flink 中对 Timestamp 的定义。与一般的认知不同，Flink 中的 Timestamp 不是一个 13 位的长整形数字，而是一个 2021-01-29T09:42:01 样式的日期。所以在使用 Flink SQL 时，直接将长整形的字段类型定义为 Timestamp 会发生异常。
